# Echoes — 十年日記

同じ日付の過去の日記をまとめて見返せる、十年日記Webアプリ。

---

## 概要

毎日の日記を書きながら、同じ月日の過去の記録（去年・一昨年…）を並べて表示します。スマホのブラウザをメイン用途に設計し、PCブラウザでも動作します。データはGoogleドライブに保存され、複数デバイス間で同期できます。

---

## 機能

- **十年日記ビュー** — 今日の入力欄と、同じ日付の過去の記録を年降順で表示
- **カレンダービュー** — 月ごとのカレンダー。日記がある日にドット表示
- **キーワード検索** — 日付・本文を横断検索
- **PINロック** — 起動時に4桁PINで保護
- **テキストインポート** — 既存日記ファイル（`=====` 区切り形式）を読み込み
- **エクスポート** — 全データをテキストファイルに書き出し
- **Google Drive同期** — `echoes-data.json` に保存。オフライン時はlocalStorageをキャッシュとして使用

## Google Drive 連携の設定方法

### 1. Google Cloud プロジェクトの作成

1. [Google Cloud Console](https://console.cloud.google.com) を開く
2. 「プロジェクトを作成」→ 任意の名前（例：`echoes-diary`）
3. 作成したプロジェクトを選択

### 2. Google Drive API の有効化

1. 左メニュー「APIとサービス」→「ライブラリ」
2. "Google Drive API" を検索して「有効にする」

### 3. OAuth 同意画面の設定

1. 「APIとサービス」→「OAuth 同意画面」
2. ユーザーの種類：**外部**
3. アプリ名・メールアドレスを入力して保存
4. スコープの追加：`.../auth/drive.file`
5. テストユーザーに自分のGoogleアカウントを追加

### 4. OAuth 2.0 クライアントIDの発行

1. 「APIとサービス」→「認証情報」→「認証情報を作成」→「OAuthクライアントID」
2. アプリケーションの種類：**ウェブアプリケーション**
3. 「承認済みのJavaScriptオリジン」に以下を追加：
   - 本番：`https://lilac3310.github.io`
   - ローカル確認用：`http://localhost:3456`
4. 「作成」→ 表示されたクライアントIDをコピー

### 5. アプリへの設定

1. Echoesを開いてPINでログイン
2. 設定 → **Google Drive 連携** → Client ID「設定」
3. コピーしたクライアントIDを貼り付けて保存
4. 「ログイン」ボタンからGoogleアカウントで認証
5. 自動的に同期が開始されます

### データ仕様

- Googleドライブのルートに `echoes-data.json` として保存
- 複数デバイスから同じGoogleアカウントでログインすると自動マージ
- マージ競合は `updatedAt` タイムスタンプで新しい方を優先

## 日記インポート形式

既存の日記データをテキストファイルでインポートできます。

```
================================
2025/04/17  令和7年
本文テキスト
================================
2024/04/17  令和6年
本文テキスト
```

- 区切り線：`=` が3文字以上並んだ行
- 日付行：`YYYY/MM/DD` 形式（元号表記は任意）
- 天気情報（晴・曇・雨・雪・霧を含む30文字未満の行）がある場合は自動認識
